## 讨论脉络

### 1. 初始发散（8 条 Agent 方向）

- 访前 Briefing、临床问答、异议 Coach、Territory 规划、政策准入、竞品情报、合规守门、经理教练
- **结论**：MVP 聚焦代表每日工作流，不做处方分析、CRM 集成、经理看板

### 2. 数据来源澄清

用户问：Briefing 中「历史互动、处方倾向、关注领域」数据如何获取？

| 数据 | 来源 |
|------|------|
| 历史互动 | 代表录入 + 访后记录 + 未来 CRM |
| 处方倾向 | **MVP 不依赖**；用进院状态、代表观察、聚合市场数据间接替代 |
| 关注领域 | 公开学术信息 + 拜访笔记 |

### 3. MVP 功能收敛

确定 Phase 1 四大模块：
1. **专科晨报 + 重磅快讯**（Daily 入口）
2. **访前 Briefing**
3. **合规问答（RAG）**
4. **访后记录（语音→结构化）**

### 4. 晨报/快讯功能扩充

用户需求：心内科代表 A 每日自动总结公众号/行业突破，并及时通知。

- 数据：RSS/官网白名单 + 公众号 Curated（MVP 不全自动抓）
- 个性化：专科标签 + 重要性打分
- 快讯门槛：重磅规则 + 日上限 2 条
- 与 Briefing 联动：「今日可聊的一句话」

### 5. 完整文档产出

用户要求总结全部讨论 → 产出 Spec + Plan + 架构 + 商业分析。

### 6. 客户端修订（v0.2）

用户明确要求：**第一版不做微信**，采用 **H5 + Web 双端**。

- 推送：Web Push（PWA）+ 站内通知，替代微信订阅消息
- 登录：手机号/邮箱，替代微信授权
- 语音：浏览器 MediaRecorder + 后端 ASR
- 微信小程序：移至 Future / Phase 3 可选

> 客户群体更多是医药代表，想针对这个群体做 agent 产品。

> 第一期应包含：按专科（如心内科）每日总结公众号/最新突破，生成晨报；网上有重大突破时及时通知。

> 总结前面所有描述，写出完整的产品设计、开发规划、技术选型。

> 第一版不做微信，预期是 H5 + Web 双端。
