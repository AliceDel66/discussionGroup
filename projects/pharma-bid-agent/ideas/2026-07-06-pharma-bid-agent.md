- **日期**：2026-07-06

---

## 主题

高校希望合作开发面向药企等场景的招标专项 Skill/Agent/Workflow。已知数据：历年招标要求 + 对应标书。需确定产品形态，并支持 A/B/C 三类招标场景。

---

## 讨论脉络

### 1. 场景澄清：药企招标 ABC 三类

| 类型 | 场景 | 典型用户 |
|------|------|----------|
| **A** | 医院/集采药品器械采购应标 | 药企招投标部、标书服务商 |
| **B** | 药企对外采购（CRO、物流、IT 等） | 采购部（发标）或供应商（应标） |
| **C** | 政企/科研类 | 高校科研处、药企 BD、产学研办 |

**结论**：ABC 全部支持，**用户新建项目时自行选择模式**。

### 2. 产品形态参谋

| 形态 | 结论 |
|------|------|
| 纯 Skill | 作模块，不单独成产品 |
| 纯 Agent | 风险高（标书胡编/废标），不做黑盒一键交稿 |
| **Workflow + Skill Pack** | **主形态**：流水线 + 人工审阅卡点 + 分模式插件 |

### 3. 架构共识

- **共享内核（~80%）**：解析、相似检索、响应矩阵、分章起草、引用溯源、合规扫描、人工审阅、导出
- **模式插件（~20%）**：A/B/C 各一套 Skill Pack（解析规则、模板、合规、索引库）
- **数据层**：历史标书按 `mode` 硬过滤检索，默认禁止跨模式混检

### 4. B 模式范围收敛

- B 含「采购方发标」与「供应商应标」两个方向
- **Phase 1**：B **仅做供应商应标**，与 A **共用应标引擎**
- **Phase 2**：B 采购方发标（RFP 生成、评分表、供应商比选）

### 5. C 模式首发文档类型

- C 可选：课题申报、产学研协议、药企合作标等，模板差异大
- **用户确认**：C 首发文档类型 = **药企合作标**
  - 典型：药企发布的研发合作、产学研联合项目、技术转移/联合实验室等招标/征集
  - 高校或 CRO/服务商作为 **应标方** 撰写合作方案

### 6. MVP 分期

| 阶段 | A（药品集采） | B（供应商） | C（药企合作标） |
|------|---------------|-------------|-----------------|
| Phase 1 | 深：解析+矩阵+检索+大纲 | 共用 A 引擎 | 中：解析+指标映射+大纲+章节初稿（**含商务/报价**） |
| Phase 2 | 分章起草+废标扫描 | 与 A 对齐加深 | 指标合规+案例库 |
| Phase 3 | 评分对齐 | — | — |
| — | — | B 采购发标 | — |

### 7. A 首发品类与 C 商务章

- **用户确认**：A 模式 Phase 1 首发品类 = **药品集采**（国采/省采；器械/服务 Phase 2 扩展）
- **用户确认**：C 药企合作标 **含商务/报价章**；AI 可生成结构与说明，**金额/费率/降幅由人工填写**

### 8. Phase 1 交付形态

- **用户确认**：**Web SaaS**（桌面浏览器为主）
- 不做：Cursor/Office 插件、H5、小程序
- 已产出：`tech/architecture.md` + `plans/plan.md` → **L3 部分可开发**
